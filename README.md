"# LSM_Compaction" 

### 项目名称：基于 LSM-Tree 架构的轻量级存储引擎系统
技术栈： Java, Java NIO , 多路归并算法, 二进制序列化

项目描述： 该项目模拟了现代键值数据库（如 RocksDB, LevelDB）核心的 LSM-tree 存储模型，实现了一个高效的 SSTable（Sorted String Table）合并与压缩（Compaction）引擎。系统能够处理磁盘上的多个无序二进制数据文件，通过多路归并、数据去重和过期数据清理，最终生成空间优化且查询高效的有序存储文件。

核心职责与技术亮点：

- 自定义二进制存储协议设计 ：设计并实现了一套紧凑的二进制 SSTable 格式，包含文件头元数据（文件大小、时间戳、键值对数量）、索引区（Key 及数据偏移量）和数据区。
- 高效多路归并排序算法 ：基于堆排序思想实现了 k 路归并算法 ，将分布在不同 SSTable 中的键值对进行全局有序排列。在排序过程中，通过时间戳（Timestamp）逻辑确保相同 Key 的数据能按版本顺序准确处理。
- 数据一致性与冗余清理 ：实现了 LSM-tree 特有的 数据去重与 Tombstone（墓碑标记）处理机制 。在合并过程中仅保留相同 Key 的最新版本，并自动剔除标记为删除（Value 为空）的数据，显著提升存储空间利用率。
- 底层 IO 与内存管理优化 ：利用 Java NIO ( ByteBuffer & FileChannel ) 进行低级别二进制数据操作。手动管理字节序（Little-Endian/Big-Endian）转换，确保在不同硬件平台下的数据一致性。
- 动态分卷存储机制 ：实现了自动化的文件切分逻辑，严格控制单个输出文件的大小（如 256KB），在保证 I/O 效率的同时，为后续的层级合并提供了标准化的物理存储基础。
